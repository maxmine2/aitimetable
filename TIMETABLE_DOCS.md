# Техническая документация: Timetable Viewer

## 1. Обзор проекта
Веб-приложение для просмотра расписания НГУ. Состоит из одного HTML-файла с встроенными CSS/JS и вспомогательного Python-сервера.

Цели:
- Быстрый рендеринг на стороне клиента без лишних зависимостей.
- Прозрачная синхронизация состояния через URL (hash) для шаринга и навигации.
- Доступные цвета и плавная анимация.
- Классификация аудиторий по местоположению.
- Рекомендация оптимальных слотов для проведения мероприятий.

### Структура файлов:
- [timetable.html](timetable.html) — Основной интерфейс и логика (все CSS/JS встроены).
- [serve.py](serve.py) — HTTP-сервер для локального запуска (необходим из-за `fetch()`).
- [nsu_data.json](nsu_data.json) — База данных (группы, преподаватели, аудитории).
- [analysis.py](analysis.py) — Скрипт-парсер для обновления данных.
- [ALGORITHM_DOCS.md](ALGORITHM_DOCS.md) — Описание алгоритма рекомендации слотов.
- [ALGORITHM_PSEUDOCODE.md](ALGORITHM_PSEUDOCODE.md) — Полный псевдокод алгоритма.
- [LOCATION_DISTINCTION.md](LOCATION_DISTINCTION.md) — Правила классификации аудиторий.
- [DATA_FORMAT.md](DATA_FORMAT.md) — Схема JSON-данных.

## 3. Режимы работы

Приложение имеет два режима, переключаемых кнопками в заголовке:

### Режим «Расписание» (`currentMode = 'schedule'`)
- Поле поиска группы с автодополнением.
- Таблица расписания 6×7 (дни × пары) с карточками занятий.
- Легенда типов занятий и чётности недель.

### Режим «Загруженность» (`currentMode = 'heatmap'`)
- Панель фильтров: факультет, специальность, ступень, курс, группа.
  - Ступень: «Бакалавриат» и «Специалитет» объединены в один пункт «Бакалавриат / Специалитет».
- Система чипов для комбинирования нескольких фильтров.
- Переключатель недели: «Всё» / «Нч» / «Чт».
- Тепловая карта загруженности с цветодевой шкалой.
- **Панель рекомендации слотов** (появляется при выборе «Нч» или «Чт»).

## 4. Программная логика (JavaScript)

### Управление состоянием
- `DATA`: Полный JSON-объект из [nsu_data.json](nsu_data.json).
- `currentMode`: `'schedule'` или `'heatmap'`.
- `currentGroup`: ID текущей группы (или `null`).
- `heatFilters`: Массив активных фильтров для тепловой карты.
- `heatWeekFilter`: `'all'` | `'odd'` | `'even'` — текущий фильтр недели.
- `lastPredictionResults`: Массив из 3 лучших слотов (или `null`).

### Ключевые функции — Расписание
1.  **`buildGroupIndex()`**: Строит индекс `[{id, label, search}]` из всех групп для автодополнения.
2.  **`selectGroup(id)`**: Загружает расписание группы, обновляет метаданные и URL.
3.  **`renderSchedule(group)`**: Генерирует HTML-таблицу 6×7 с карточками занятий.
4.  **`renderCard(c)`**: Возвращает HTML одной карточки занятия (тип, предмет, преподаватель, аудитория).

### Ключевые функции — Тепловая карта
5.  **`switchMode(mode)`**: Переключает UI между расписанием и загруженностью.
6.  **`initHeatmapControls()`**: Инициализирует каскадные select’ы фильтров (факультет → специальность → группа). При инициализации объединяет «Бакалавриат» и «Специалитет» в единый пункт ступени.
7.  **`updateHfDependents()`**: Обновляет зависимые списки при изменении фильтра. Использует `degreeMatches()` для сопоставления объединённой ступени.
8.  **`addFilter()` / `removeFilter(idx)`**: Управление массивом `heatFilters`.
9.  **`renderChips()`**: Отображает чипы текущих фильтров.
10. **`setWeekFilter(val)`**: Устанавливает фильтр недели, показывает/скрывает панель рекомендации, перерисовывает тепловую карту.
11. **`collectHeatmapGroups()`**: Собирает массив объектов групп по текущим фильтрам. Использует `degreeMatches()` для сопоставления ступени.
12. **`renderHeatmap()`**: Считает занятость по слотам, генерирует таблицу с цветовой шкалой. После отрисовки — повторно применяет подсветку и результаты рекомендации (если есть).
13. **`heatColor(pct)`**: Интерполяция HSL (голубой → тёмно-синий) по проценту заполнения.

### Ключевые функции — Классификация аудиторий
14. **`classifyRoom(roomId)`**: Определяет группу местоположения аудитории (1–5) по 11 правилам приоритета из [LOCATION_DISTINCTION.md](LOCATION_DISTINCTION.md).

| Группа | Название     | Примеры ID                                          |
|--------|-------------|-----------------------------------------------------|
| 1      | НК/КПА       | `nk_5_301_798`, `Ауд. 209 КПА`, `Ауд. 5207`      |
| 2      | ГК/ЛК        | `gk_2_675_621`, `lk_...`, `au_...`, `...ГК...`     |
| 3      | Онлайн        | `...Google Meet...`, `...BigBlueButton...`           |
| 4      | Кампус        | `...МНОЦ...`, `Ауд. Спортивный комплекс`, `...Пирогова...` |
| 5      | Вне/Неясно    | `...Институт...`, `...ФГБУН...`, всё остальное      |

### Ключевые функции — Рекомендация слотов
15. **`generateCandidates(allowedDays, morningCutoff, eveningCutoff)`**: Генерирует все пары `(день, слот)` в заданном диапазоне.
16. **`getClassesToday(group, day, weekParity)`**: Фильтрует расписание группы по дню и чётности недели.
17. **`computeL(refLocGroup, eventLocation)`**: Вычисляет фактор близости местоположения (0.0–1.0).
18. **`scoreGroup(group, day, eventSlot, weekParity, eventLocation)`**: Полный скоринг одной группы для одного кандидатного слота. Возвращает `{bucket, score, detail}`, где `detail` содержит все промежуточные значения расчёта (reason, classesToday, campusFactor, P, L, S, D, rawScore и др.).
19. **`findBestEventSlots(groupsDict, weekParity, morningCutoff, eveningCutoff, eventLocation, allowedDays)`**: Ранжирует все кандидатные слоты и возвращает топ-3.
20. **`collectHeatmapGroupIds()`**: Аналог `collectHeatmapGroups()`, но возвращает `{groupId: groupObj}` словарь для алгоритма.
21. **`runPrediction()`**: Обработчик кнопки «Рассчитать» — собирает параметры из UI, вызывает алгоритм, перерисовывает тепловую карту с подсветкой.
22. **`renderPredictionResults(top3)`**: Генерирует HTML трёх карточек результатов. Строки статистики (более/менее вероятные, занятые) кликабельны и открывают модальное окно.
23. **`openGroupsModal(rankIdx, tab)`**: Открывает модальное окно для выбранного слота. Заполняет заголовок, строит вкладки (more/less/busy) и дерево групп через `buildGroupTree()`.
24. **`switchModalTab(tabKey)`**: Переключает активную вкладку внутри модалки и скрывает карточку подробностей.
25. **`buildGroupTree(groupIds, chipCls, rankIdx)`**: Группирует массив ID групп в иерархию «Факультет → Специальность → Группы». Внутри каждой специальности группы отсортированы по ступени, затем по курсу, с инлайн-метками `.gm-dy-sep` (напр. «Бак 2к») при смене ступени/курса. Параметр `rankIdx` передаётся чипам для доступа к данным скоринга.
26. **`toggleFaculty(header)`**: Сворачивает/разворачивает блок факультета в дереве.
27. **`showGroupDetail(gid, rankIdx, event)`**: При клике на чип группы показывает карточку с метаданными и полной детализацией скоринга (компоненты P, L, S, D, множители, формулу и итоговый балл) из `lastPredictionResults[rankIdx].groupDetails[gid]`.
28. **`closeGroupsModal()`**: Закрывает модальное окно.
29. **`highlightPredictedSlots(top3)`**: Добавляет CSS-классы и значки рангов к ячейкам тепловой карты.

### Ключевые функции — Пресеты
30. **`getPresets()` / `setPresets(presets)`**: Чтение/запись словаря пресетов в `localStorage` (ключ `timetable_presets`).
31. **`refreshPresetSelect()`**: Обновляет `<select>` со списком сохранённых пресетов.
32. **`gatherPredConfig()`**: Собирает текущие настройки (место, пары, дни, фильтры, неделя) в объект.
33. **`applyPredConfig(cfg)`**: Применяет конфигурацию: восстанавливает фильтры, неделю, контролы.
34. **`savePreset()`**: Запрашивает имя и сохраняет текущую конфигурацию в `localStorage`.
35. **`loadPreset()`**: Загружает выбранный пресет и применяет его настройки.
36. **`deletePreset()`**: Удаляет выбранный пресет после подтверждения.

### Ключевые функции — Шаринг результатов
37. **`encodeResults()`**: Кодирует результаты и настройки в компактную base64url-строку (JSON → UTF-8 → base64url).
38. **`decodeResults(code)`**: Декодирует base64url-код обратно в объект.
39. **`exportShareCode()`**: Кодирует результаты и копирует код в буфер обмена + отображает его.
40. **`importShareCode()`**: Читает код из поля ввода, декодирует. Создаёт синтетические результаты (без деталей групп, модалка недоступна) и отображает карточки.

### Утилиты
41. **`esc(s)`**: Экранирование HTML-сущностей.
42. **`saveHash()` / `restoreFromHash()`**: Сериализация/десериализация состояния в URL-хэш.
43. **`degreeMatches(filterDeg, groupDeg)`**: Сопоставляет значение фильтра ступени со ступенью группы. Если фильтр = `COMBINED_BAK_SPEC` («Бакалавриат / Специалитет»), соответствует обеим ступеням.

## 5. Алгоритм скоринга (`scoreGroup`)

Подробная документация: [ALGORITHM_DOCS.md](ALGORITHM_DOCS.md), [ALGORITHM_PSEUDOCODE.md](ALGORITHM_PSEUDOCODE.md).

### Константы

```js
W_PROXIMITY  = 3.0    // вес фактора близости
W_LOCATION   = 3.0    // вес фактора корпуса
W_SANDWICH   = 2.0    // вес фактора «бутерброда»
W_DEPARTURE  = 2.0    // вес штрафа за отъезд
BUCKET_THRESHOLD = 4.0 // порог «скорее придёт»
PROXIMITY_DECAY = { 1: 1.0, 2: 0.55, 3: 0.25, 4: 0.1 } // зазор ≥ 5 → 0
LATE_HOUR_FACTOR = { 1–4: 1.0, 5: 0.95, 6: 0.85, 7: 0.7 }
TIREDNESS_TABLE  = { ≤1: 1.0, 2: 0.95, 3: 0.9, 4: 0.85, ≥5: 0.8 }
```

### Этапы (на каждую группу × кандидатный слот)

1. **Конфликт** — обязательное занятие → `have_class`; факультатив → мягкий конфликт (штраф −1.0).
2. **Пустой день** — нет занятий → `less_likely`, score = 0.
3. **Присутствие на кампусе** — множитель `campusFactor`: ≥2 якорных → ×1.0, 1 → ×0.6, только онлайн → ×0.15, только вне → ×0.1.
4. **Близость (P)** — расстояние до ближайшего кампусного занятия (1→1.0, 2→0.55, 3→0.25, 4→0.1, ≥5→0). Модификатор направления: до первого ×0.6, после последнего ×0.8, между ×1.0.
5. **Корпус (L)** — совпадение корпуса с мероприятием (1.0→тот же, 0.75→НК↔ГК, 0.5→кампус, 0.2→онлайн, 0→вне).
6. **Бутерброд (S)** — мероприятие между двумя занятиями: оба якорных → 1.0, якорь+онлайн → 0.5, иначе → 0.
7. **Штраф отъезда (D)** — следующее занятие вне кампуса: через 1 слот → −2.0, через 2 → −1.0.
8. **Усталость (T)** — накопленная нагрузка от предшествующих пар. `effective_load = пары_до − 0.5 × пропуски`. Множитель: ≤1 → ×1.0, 2 → ×0.95, 3 → ×0.9, 4 → ×0.85, ≥5 → ×0.8.
9. **Вечерний фактор** — снижение доступности транспорта вечером. Слоты 1–4 → ×1.0, 5 → ×0.95, 6 → ×0.85, 7 → ×0.7.
10. **Итого**: `raw = 3P + 3L + 2S + D` (clamped ≥0), `score = raw × campusFactor × tirednessFactor × lateHourFactor − softConflictPenalty`.
11. **Бакет**: score ≥ 4.0 → `more_likely`, иначе → `less_likely`.

### Ранжирование слотов

Все кандидатные `(день, слот)` сортируются по:
1. Суммарный балл (убывание).
2. Количество `more_likely` (убывание).
3. Количество `have_class` (возрастание).

Возвращаются 3 лучших.

## 6. Граф вызовов (Call Graph)

### Инициализация
```
fetch('nsu_data.json') → buildGroupIndex() → restoreFromHash()
  └─ restoreFromHash:
     ├─ mode=schedule → selectGroup(gid) → renderSchedule()
     └─ mode=heatmap  → switchMode('heatmap') → initHeatmapControls()
        ├─ restore filters → renderChips()
        ├─ setWeekFilter(val) → renderHeatmap()
        └─ restore pred → runPrediction()
```

### Режим расписания
```
selectGroup(gid) → renderSchedule(group) → renderCard(c) ×N
                 → saveHash()
```

### Режим загруженности
```
addFilter() / removeFilter(idx) → renderChips() + renderHeatmap() + saveHash()
setWeekFilter(val) → show/hide prediction-panel → renderHeatmap() → saveHash()
renderHeatmap() → collectHeatmapGroups() → build table → container.innerHTML
  └─ if lastPredictionResults:
     ├─ highlightPredictedSlots(top3)
     └─ renderPredictionResults(top3)
```

### Рекомендация слотов
```
runPrediction() → collectHeatmapGroupIds()
               → findBestEventSlots()
                  ├─ generateCandidates()
                  └─ for each (day,slot) × group:
                     └─ scoreGroup()
                        ├─ getClassesToday()
                        ├─ classifyRoom() ×N
                        ├─ computeL()
                        └─ → {bucket, score, detail}
               → lastPredictionResults = top3 (each with groupDetails dict)
               → renderHeatmap() (re-renders with highlights)
               → saveHash()
```

### Модальное окно групп
```
click .stat-row.clickable → openGroupsModal(rankIdx, tab)
  ├─ gm-header: день + пара
  ├─ gm-tabs: more | less | busy  → switchModalTab(tabKey)
  └─ buildGroupTree(groupIds, chipCls, rankIdx)
     └─ for each faculty:
        └─ for each specialty:
           └─ gm-group-chip × N → click → showGroupDetail(gid, rankIdx)
              └─ gm-detail-card (метаданные + разбор скоринга из groupDetails)
click .gm-close / overlay → closeGroupsModal()
```

### Пресеты
```
savePreset() → gatherPredConfig() → localStorage.setItem('timetable_presets')
loadPreset() → localStorage.getItem → applyPredConfig(cfg)
  ├─ heatFilters = cfg.filters
  ├─ renderChips()
  ├─ setWeekFilter(cfg.week)
  └─ восстановить контролы (location, morning, evening, days)
deletePreset() → localStorage update → refreshPresetSelect()
```

### Шаринг результатов
```
exportShareCode() → encodeResults()
  └─ payload {v, w, cfg, r[]} → JSON → UTF-8 → base64url
importShareCode() → decodeResults(code)
  └─ base64url → JSON → синтетические lastPredictionResults
  └─ renderHeatmap()
```

## 7. Панель рекомендации слотов (UI)

### Условие отображения
Панель `#prediction-panel` видна **только** когда:
- Активен режим «Загруженность» (`currentMode = 'heatmap'`).
- Выбрана конкретная неделя: «Нч» (`heatWeekFilter = 'odd'`) или «Чт» (`heatWeekFilter = 'even'`).

При переключении на «Всё» панель скрывается, `lastPredictionResults` сбрасывается.

### Сброс результатов
`lastPredictionResults` автоматически сбрасывается при:
- Добавлении фильтра (`addFilter()`).
- Удалении фильтра (`removeFilter()`).
- Смене недели (`setWeekFilter()`) — любое изменение, включая переключение между «Нч» и «Чт».

### Элементы управления
Панель имеет эмодзи-заголовок и секционные метки (`.pred-section-label`): 📍 Место, 🕐 Пары, 📅 Дни.

| Место мероприятия | `#pred-location` | `1` = НК/КПА, `2` = ГК/ЛК |

- `eveningCutoff < morningCutoff` → alert.

При клике на строку статистики открывается центрированное модальное окно с:

## 8. Управление состоянием (State Management)

### Переменные состояния (глобальные)

```js
let DATA = null;                    // весь nsu_data.json
let groupIndex = [];                // [{id, label, search}, ...]
let currentGroup = null;            // ID текущей группы
let currentMode = 'schedule';       // 'schedule' | 'heatmap'
let heatFilters = [];               // [{faculty, specialty, degree, year, group, label}]
let heatWeekFilter = 'all';         // 'all' | 'odd' | 'even'
let heatmapInited = false;          // флаг инициализации контролов
let lastPredictionResults = null;   // Array<{day, slot, totalScore, groupDetails, ...}> | null
```

### Правила
- Состояние синхронизируется с URL через `saveHash()`/`restoreFromHash()`.
- `renderHeatmap()` автоматически восстанавливает подсветку и результаты рекомендации из `lastPredictionResults`.
- `lastPredictionResults` сбрасывается при любом изменении фильтров или смене недели.

## 9. Формат хэша URL (URL Hash Format)

Хэш кодируется как URL-encoded JSON:

```
#%7B"mode":"schedule","group":"24812.1"%7D
#%7B"mode":"heatmap","filters":[{"f":"16"}],"week":"odd","pred":{"loc":"1","m":"3","e":"6","days":["1","2","3","4","5"]}%7D
```

### Поля объекта состояния

| Поле | Тип | Описание |
|------|-----|---------|
| `mode` | string | `"schedule"` или `"heatmap"` |
| `group` | string | ID группы (только в mode=schedule) |
| `filters` | array | Массив фильтров `{f, s, d, y, g}` (только в mode=heatmap) |
| `week` | string | `"odd"` или `"even"` (опционально, по умолчанию `"all"`) |
| `pred` | object | Параметры рекомендации (только если активна) |
| `pred.loc` | string | `"1"` (НК/КПА) или `"2"` (ГК/ЛК) |
| `pred.m` | string | Первый допустимый слот (`"1"`–`"7"`) |
| `pred.e` | string | Последний допустимый слот (`"1"`–`"7"`) |
| `pred.days` | string[] | Допустимые дни (`["1","2","3","4","5"]`) |

При восстановлении из хэша с `pred` + `week` автоматически запускается `runPrediction()`.

## 10. Формат данных (JSON)
- `bell_schedule`: Время начала и конца каждой пары (слоты 1–7).
- `teachers` / `rooms`: Справочники ID → Название.
- `groups`: Основной объект, где ключи — номера групп. Каждая содержит массив `schedule` с деталями занятий.

Подробная схема: [DATA_FORMAT.md](DATA_FORMAT.md).

## 11. Запуск
1. Убедитесь, что [nsu_data.json](nsu_data.json) находится в той же папке.
2. Запустите `python serve.py`.
3. Откройте `http://localhost:8765/timetable.html`.

### Использование рекомендации слотов
1. Перейдите в режим «Загруженность».
2. Добавьте фильтры (факультет, курс и т.д.) или оставьте пустыми для анализа всех групп.
3. Выберите неделю: «Нч» или «Чт» — появится панель «Рекомендация слотов для мероприятия».
4. Укажите место проведения, допустимый диапазон пар и дни.
5. Нажмите «Рассчитать».
6. Результаты отобразятся под тепловой картой, а лучшие слоты подсветятся в таблице.

### Сохранение настроек как пресета
1. Настройте фильтры, неделю и параметры рекомендации.
2. Нажмите «💾 Сохранить», введите имя.
3. Пресет появится в выпадающем списке.
4. Выберите пресет и нажмите «📂 Загрузить» для восстановления.

### Поделиться результатами
1. После расчёта нажмите «📋 Копировать код» — компактная строка скопируется в буфер.
2. Передайте код другому пользователю.
3. Получатель вставляет код в поле и нажимает «📥 Импорт» — результаты отобразятся (карточки + подсветка тепловой карты, но без детализации групп).
