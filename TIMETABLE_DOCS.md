# Техническая документация: Timetable Viewer

## Обзор проекта
Веб-приложение для просмотра расписания НГУ. Состоит из одного HTML-файла с встроенными CSS/JS и вспомогательного Python-сервера.

Цели:
- Быстрый рендеринг на стороне клиента без лишних зависимостей.
- Прозрачная синхронизация состояния через URL (hash) для шаринга и навигации.
- Доступные цвета и плавная анимация.
- Классификация аудиторий по местоположению.
- Рекомендация оптимальных слотов для проведения мероприятий.

### Структура файлов:
- [timetable.html](timetable.html) — Основной интерфейс и логика (все CSS/JS встроены).
- [serve.py](serve.py) — HTTP-сервер для локального запуска (необходим из-за `fetch()`).
- [nsu_data.json](nsu_data.json) — База данных (группы, преподаватели, аудитории).
- [analysis.py](analysis.py) — Скрипт-парсер для обновления данных.
- [ALGORITHM_DOCS.md](ALGORITHM_DOCS.md) — Описание алгоритма рекомендации слотов.
- [ALGORITHM_PSEUDOCODE.md](ALGORITHM_PSEUDOCODE.md) — Полный псевдокод алгоритма.
- [LOCATION_DISTINCTION.md](LOCATION_DISTINCTION.md) — Правила классификации аудиторий.
- [DATA_FORMAT.md](DATA_FORMAT.md) — Схема JSON-данных.

## Режимы работы

Приложение имеет два режима, переключаемых кнопками в заголовке:

### Режим «Расписание» (`currentMode = 'schedule'`)
- Поле поиска группы с автодополнением.
- Таблица расписания 6×7 (дни × пары) с карточками занятий.
- Легенда типов занятий и чётности недель.

### Режим «Загруженность» (`currentMode = 'heatmap'`)
- Панель фильтров: факультет, специальность (фиксированная ширина 260px, не прыгает при смене факультета), ступень, курс, группа.
  - Ступень: «Бакалавриат» и «Специалитет» объединены в один пункт «Бакалавриат / Специалитет».
- Система чипов для комбинирования нескольких фильтров.
- Переключатель недели: «Всё» / «Нч» / «Чт».
- Тепловая карта загруженности с цветодевой шкалой. **Таблица центрирована** (`max-width: 1000px`), пресеты расположены справа.
- **Панель рекомендации слотов** (появляется при выборе «Нч» или «Чт»).
- **Боковая панель пресетов** (`.preset-sidebar`, 300px) — правая колонка с sticky-позиционированием. На мобильных (≤900px) перемещается под таблицу.

## Программная логика (JavaScript)

### Управление состоянием
- `DATA`: Полный JSON-объект из [nsu_data.json](nsu_data.json).
- `currentMode`: `'schedule'` или `'heatmap'`.
- `currentGroup`: ID текущей группы (или `null`).
- `heatFilters`: Массив активных фильтров для тепловой карты.
- `heatWeekFilter`: `'all'` | `'odd'` | `'even'` — текущий фильтр недели.
- `lastPredictionResults`: Массив из 3 лучших слотов (или `null`). Содержит только результаты алгоритма `findBestEventSlots`.
- `_extraResults`: Словарь дополнительных результатов: ключ `100` — результат клика по произвольной ячейке, ключи `50–55` — результаты compare-слотов. Очищается при сбросе `lastPredictionResults`.
- `getResult(idx)`: Хелпер-функция, ищет результат сначала в `_extraResults`, затем в `lastPredictionResults`. Используется всеми функциями модального окна (`openGroupsModal`, `renderAnchorBar`, `renderFatigueHistogram` и др.).
- `_lastAudienceDiff`: Последний рассчитанный diff между двумя слотами. Содержит `{onlyA, onlyB, both, resA, resB}` — группы только в слоте A, только в B и в обоих.
- `_pairwiseSelection`: Массив ключей результатов (макс. 2) для попарного сравнения. Ключи: `0`–`2` (prediction top-3), `50`–`55` (pinned слоты), `100` (клик по ячейке). Очищается при `runPrediction()` и `clearCompare()`.
- `compareMode`: Флаг режима отметки слотов (pin mode). Клик по ячейке добавляет/убирает слот. Кнопка UI: «📌 Отметить слоты».
- `compareSlots`: Массив выбранных слотов `[{day, slot, color}]` (макс. 6).
- `compareResults`: Массив посчитанных результатов для compare-слотов.
- `COMPARE_COLORS`: 6 ярких, контрастных цветов для compare-слотов: vivid red (`#e6194b`), bright orange (`#f58231`), yellow (`#ffe119`), purple (`#911eb4`), magenta (`#f032e6`), cyan (`#42d4f4`).
- `COMPARE_TEXT`: Массив цветов текста для каждого compare-цвета (белый или тёмный для контрастности).

#### ⚠️ Правило: Адаптация состояния
> Любая новая фича, добавляющая пользовательское состояние (переключатели, выбранные элементы и т.д.),
> **ОБЯЗАНА** быть представлена во всех трёх механизмах сохранения:
> 1. **URL-хеш** (`saveHash()` / `restoreFromHash()`) — для закладок и навигации.
> 2. **Пресеты** (`gatherFullState()` / `applyFullState()`) — для localStorage-пресетов.
> 3. **Шеринг** (`encodeResults()` / `importShareCode()`) — для передачи между пользователями.
>
> Все три используют единый формат через `gatherFullState()` / `applyFullState()`.
> При добавлении нового поля — добавить его в `gatherFullState()`, в `applyFullState()`, в `saveHash()`,
> в `restoreFromHash()`, в `encodeResults()`, и в `importShareCode()`.

### Ключевые функции — Расписание
1.  **`buildGroupIndex()`**: Строит индекс `[{id, label, search}]` из всех групп для автодополнения.
2.  **`selectGroup(id)`**: Загружает расписание группы, обновляет метаданные и URL.
3.  **`buildSlotMap(schedule, opts)`**: Универсальный хелпер для построения словаря `ключ → [записи]` из массива расписания. Параметры: `opts.day` — фильтр по одному дню (ключи становятся `"slot"`), `opts.weekParity` — фильтр по чётности (`'odd'|'even'|'all'`). Без фильтра дня ключи имеют формат `"day-slot"`. Используется в `renderSchedule()` и `renderMiniDaySchedule()`.
4.  **`renderClassEntry(c, mini)`**: Универсальный рендерер одной записи занятия. При `mini=false` возвращает полную карточку `.class-card` (тип, предмет, преподаватель, аудитория, тег недели). При `mini=true` — компактную карточку `.mini-card` для мини-расписания дня. Используется в `renderSchedule()` и `renderMiniDaySchedule()`.
5.  **`renderSchedule(group)`**: Генерирует HTML-таблицу 6×7 с карточками занятий. Использует `buildSlotMap()` и `renderClassEntry(c, false)`.
6.  **`renderCard(c)`**: Обёртка-алиас для обратной совместимости, делегирует в `renderClassEntry(c, false)`.

### Ключевые функции — Тепловая карта
7.  **`switchMode(mode)`**: Переключает UI между расписанием и загруженностью.
8.  **`initHeatmapControls()`**: Инициализирует каскадные select’ы фильтров (факультет → специальность → группа). При инициализации объединяет «Бакалавриат» и «Специалитет» в единый пункт ступени.
9.  **`updateHfDependents()`**: Обновляет зависимые списки при изменении фильтра. Использует `degreeMatches()` для сопоставления объединённой ступени.
10.  **`addFilter()` / `removeFilter(idx)`**: Управление массивом `heatFilters`.
11.  **`renderChips()`**: Отображает чипы текущих фильтров.
12. **`setWeekFilter(val)`**: Устанавливает фильтр недели, показывает/скрывает панель рекомендации, перерисовывает тепловую карту.
13. **`collectHeatmapGroups()`**: Собирает массив объектов групп по текущим фильтрам. Использует `degreeMatches()` для сопоставления ступени.
14. **`renderHeatmap()`**: Считает занятость по слотам, генерирует таблицу с цветовой шкалой. Все ячейки кликабельны: в обычном режиме `onclick` вызывает `handleCellClick(day, slot)`, в режиме отметки (pin mode) — `toggleCompareSlot()`. После отрисовки — повторно применяет подсветку, результаты рекомендации и compare-слоты.
15. **`heatColor(pct)`**: Интерполяция HSL (голубой → тёмно-синий) по проценту заполнения.

### Ключевые функции — Классификация аудиторий
16. **`classifyRoom(roomId)`**: Определяет группу местоположения аудитории (1–5) по 11 правилам приоритета из [LOCATION_DISTINCTION.md](LOCATION_DISTINCTION.md).

| 1      | НК/КПА       | `nk_5_301_798`, `Ауд. 209 КПА`, `Ауд. 5207`      |
| 2      | ГК/ЛК        | `gk_2_675_621`, `lk_...`, `au_...`, `...ГК...`     |
| 3      | Онлайн        | `...Google Meet...`, `...BigBlueButton...`           |
| 4      | Кампус        | `...МНОЦ...`, `Ауд. Спортивный комплекс`, `...Пирогова...` |
| 5      | Вне/Неясно    | `...Институт...`, `...ФГБУН...`, всё остальное      |

### Ключевые функции — Рекомендация слотов
17. **`generateCandidates(allowedDays, morningCutoff, eveningCutoff)`**: Генерирует все пары `(день, слот)` в заданном диапазоне.
18. **`getClassesToday(group, day, weekParity)`**: Фильтрует расписание группы по дню и чётности недели.
19. **`computeL(refLocGroup, eventLocation)`**: Вычисляет фактор близости местоположения (0.0–1.0).
20. **`scoreGroup(group, day, eventSlot, weekParity, eventLocation)`**: Полный скоринг одной группы для одного кандидатного слота. Возвращает `{bucket, score, detail}`, где `detail` содержит все промежуточные значения расчёта (reason, classesToday, campusFactor, P, L, S, D, rawScore и др.).
21. **`findBestEventSlots(groupsDict, weekParity, morningCutoff, eveningCutoff, eventLocation, allowedDays)`**: Ранжирует все кандидатные слоты и возвращает топ-3.
22. **`collectHeatmapGroupIds()`**: Аналог `collectHeatmapGroups()`, но возвращает `{groupId: groupObj}` словарь для алгоритма.
23. **`runPrediction()`**: Обработчик кнопки «Рассчитать» — собирает параметры из UI, вызывает алгоритм, перерисовывает тепловую карту с подсветкой. Очищает `_pairwiseSelection`.
24. **`renderPredictionResults(top3)`**: Генерирует HTML трёх карточек результатов. Строки статистики (более/менее вероятные, занятые) кликабельны и открывают модальное окно. Если в слоте нет групп «Скорее придут» (`moreLikely.length === 0`), под строкой отображается курсивное уведомление. Каждая карточка также включает диаграмму «Откуда придут студенты» и гистограмму «Усталость аудитории». После карточек вызывает `renderCompareResults()` для отображения сравниваемых слотов.
25. **`openGroupsModal(rankIdx, tab)`**: Открывает модальное окно для выбранного слота. Заполняет левую колонку (`.gm-left`) двумя вкладками: «Паттерны» и «Группы». Вкладка «Группы» содержит три секции с цветными заголовками («Скорее придут»/«Вряд ли»/«Занятия»), каждая со своим `buildGroupTree()`. Правая колонка сбрасывается в placeholder. Параметр `tab` принимает `'patterns'` или любое другое значение (нормализуется в `'groups'`).
26. **`switchModalTab(tabKey, rankIdx)`**: Переключает активную вкладку внутри модалки. При переключении на «Паттерны» заполняет правую колонку через `buildPatternTabDetail()`.
27. **`buildGroupTree(groupIds, chipCls, rankIdx)`**: Группирует массив ID групп в иерархию «Факультет → Специальность → Группы». Внутри каждой специальности группы отсортированы по ступени, затем по курсу, с инлайн-метками `.gm-dy-sep` (напр. «Бак 2к») при смене ступени/курса. Параметр `rankIdx` передаётся чипам для доступа к данным скоринга.
28. **`toggleFaculty(header)`**: Сворачивает/разворачивает блок факультета в дереве.
29. **`showGroupDetail(gid, rankIdx, event)`**: При клике на чип группы заполняет правую колонку (`.gm-right`) карточкой с метаданными, мини-расписанием дня (с travel-строками) и полной детализацией скоринга (компоненты P, L, S, D, множители, формулу и итоговый балл) из `getResult(rankIdx).groupDetails[gid]`.
30. **`renderMiniDaySchedule(gid, rankIdx)`**: Генерирует компактную HTML-таблицу расписания группы `gid` на день рекомендованного слота. Включает:
    - Travel-строки (`tr.mini-travel-row`) между слотами при смене корпуса (используя `classifyRoom()`).
    - Информацию о прибытии (`← из [корпус]`) в слоте мероприятия (из `detail.refClass`).
    - Подсветку слота мероприятия и конфликтных слотов.
31. **`closeGroupsModal()`**: Закрывает модальное окно.
32. **`buildPatternTab(rankIdx)`**: Строит содержимое вкладки «Паттерны»: кластеризует группы по набору занятых слотов, сортирует по размеру кластера, отображает до 10 паттернов. Чекбокс `patternIncludeLess` позволяет включить группы «Вряд ли придут» в кластеризацию. Строки кликабельны → `showPatternDetail()`. Чипы раскрываемые (первый ряд по умолчанию, >6 — кнопка «ещё»).
33. **`buildPatternTabDetail(rankIdx)`**: Генерирует содержимое правой колонки когда активна вкладка «Паттерны»: включает диаграмму `renderAnchorBar()` и гистограмму `renderFatigueHistogram()`.
34. **`renderAnchorBar(rankIdx)`**: Инструмент «Откуда придут студенты» — stacked bar chart, показывающий из какого корпуса приходят группы `moreLikely` (на основе `detail.refClass.room_id → classifyRoom()`). Сегменты окрашены по зонам: НК/КПА, ГК/ЛК, Онлайн, Кампус, Вне.
35a. **`rebuildPatternTab(rankIdx)`**: Пересобирает содержимое вкладки «Паттерны» при изменении чекбокса `patternIncludeLess`.
35b. **`showPatternDetail(key, rankIdx)`**: Показывает агрегированный мини-день для кластера с ключом `key` в правой колонке. Использует те же CSS-стили что и групповой мини-день (`.mini-day-schedule`, `.mini-day-table`, `.mini-event-slot`, `.mini-travel-row`). Вместо послотовых origin-flow баров, размещает одну травел-строку с агрегированными данными о происхождении непосредственно перед слотом мероприятия (аналогично указателям «Корпус X → Корпус Y» в групповом мини-дне).
35c. **`renderPatternAnchorBar(gids, r)`**: Аналог `renderAnchorBar()`, но для произвольного подмножества групп `gids` вместо всех `moreLikely`.
36. **`renderFatigueHistogram(rankIdx)`**: Инструмент «Усталость аудитории» — гистограмма из 5 столбиков (0, 1, 2, 3, 4+ пар до мероприятия), показывающая распределение `detail.effectiveLoad` среди групп `moreLikely`. Помогает оценить, насколько «свежая» аудитория ожидается.
37. **`highlightPredictedSlots(top3)`**: Добавляет CSS-классы, значки рангов к ячейкам тепловой карты. Вызывает `highlightCompareSlots()` для подсветки compare-слотов.

### Ключевые функции — Кликабельные ячейки и Сравнение
38. **`handleCellClick(day, slot)`**: Универсальный обработчик клика по ячейке тепловой карты. В режиме отметки (`compareMode`, pin mode) вызывает `toggleCompareSlot()`. В обычном режиме — считает статистику для данного day/slot через `scoreGroup()` по всем группам, создаёт синтетический результат и открывает модальное окно.
39. **`toggleCompareMode()`**: Переключает режим отметки слотов (pin mode). Обновляет кнопку «📌 Отметить слоты» / «📌 Выбор (вкл)» и отображает чипы выбранных слотов.
40. **`toggleCompareSlot(day, slot)`**: Добавляет или убирает слот из `compareSlots` (макс. 6). Пересчитывает результаты и перерисовывает тепловую карту.
41. **`removeCompareSlot(idx)`**: Удаляет слот по индексу, переназначает цвета, перерисовывает.
42. **`clearCompare()`**: Полностью очищает compare-слоты и `_pairwiseSelection`.
43. **`recomputeCompareResults()`**: Скорит все compare-слоты через `scoreGroup()`, сохраняет в `compareResults`.
44. **`highlightCompareSlots()`**: Добавляет CSS-класс `compare-sel` с CSS-переменной `--cmp-color` и нумерованный кружок `.cmp-badge` (порядковый номер, цвет фона из `COMPARE_COLORS`, цвет текста из `COMPARE_TEXT`) к ячейкам тепловой карты.
45. **`renderCompareChips()`**: Отображает цветные чипы выбранных слотов под панелью рекомендации.
46. **`renderCompareResults()`**: Генерирует секцию `.compare-results-section` (заголовок «📌 Отмеченные слоты») с карточками отмеченных слотов, расположенными по 3 в ряд. Каждая карточка включает статистику, anchor bar, fatigue histogram и кнопку `.card-compare-btn` для попарного сравнения. Заголовки окрашены в цвет слота.

### Ключевые функции — Попарное сравнение слотов
47. **`computeAudienceDiff(resA, resB)`**: Сравнивает два результата и классифицирует все группы в три категории: `onlyA` (в `moreLikely` у A, но не у B), `onlyB` (в `moreLikely` у B, но не у A), `both` (в `moreLikely` у обоих). Сохраняет результат в `_lastAudienceDiff`.
48. **`togglePairwiseSelect(resultKey)`**: Переключает выбор карточки для попарного сравнения. Добавляет/убирает ключ из `_pairwiseSelection` (макс. 2). Обновляет CSS-класс `.selected` на кнопке `.card-compare-btn` и вызывает `renderPairwiseBar()`.
49. **`updatePairwiseButtons()`**: Обновляет визуальное состояние (`.selected`) всех кнопок `.card-compare-btn` в соответствии с текущим `_pairwiseSelection`.
50. **`renderPairwiseBar()`**: Отображает/скрывает плавающую панель `.pairwise-compare-bar` внизу экрана. При 0 выбранных — скрывает. При 1 — показывает «Выберите второй слот…». При 2 — показывает кнопку «Сравнить» (`.pairwise-run`) и «Отмена» (`.pairwise-cancel`).
51. **`clearPairwiseSelection()`**: Очищает `_pairwiseSelection`, снимает `.selected` со всех кнопок, скрывает плавающую панель.
52. **`openPairwiseComparison()`**: Вызывается при клике «Сравнить» в плавающей панели. Получает два результата через `getResult()`, вызывает `computeAudienceDiff()` и `openDiffModal()`. После открытия модалки очищает попарный выбор.
53. **`openDiffModal()`**: Открывает трёхколоночное модальное окно «Сравнение слотов» на основе `_lastAudienceDiff`. Колонки: «Только в слоте A» (синий), «В обоих слотах» (зелёный, центр), «Только в слоте B» (красный). Каждая колонка содержит итоговую статистику и список метагрупп через `renderMetagroupList()`.
54. **`closeDiffModal()`**: Закрывает модальное окно сравнения.
55. **`buildMetagroups(groupIds)`**: Агрегирует массив ID групп в метагруппы по формуле «Факультет — Ступень Курс» (напр. «ФИТ — Бак 2к»). Возвращает отсортированный массив `[{label, groups}]` — от самой большой метагруппы к самой маленькой.
56. **`renderMetagroupList(metagroups, colorCls, chipCls)`**: Рендерит список сворачиваемых карточек `.dm-metagroup` с цветовым акцентом (`colorCls`: `.dm-mg-green`/`.dm-mg-blue`/`.dm-mg-red`). Каждая карточка содержит заголовок с названием метагруппы и счётчиком, и тело с чипами отдельных групп.

### Ключевые функции — Единое управление состоянием
57. **`gatherFullState()`**: Собирает полное состояние heatmap-режима в единый объект: `{loc, m, e, days, filters, week, cmp}`. Поле `cmp` — массив `[{d, s}]` compare-слотов (опционально). Используется пресетами, URL-хешем и шерингом.
58. **`applyFullState(cfg, opts)`**: Применяет полное состояние: восстанавливает фильтры, неделю, контролы, compare-слоты. `opts.runPrediction` — запускает `runPrediction()` после восстановления. Пересчитывает `compareResults` если есть compare-слоты.
59. **`gatherPredConfig()` / `applyPredConfig(cfg)`**: Legacy-алиасы для `gatherFullState()` / `applyFullState()`.

### Ключевые функции — Пресеты
60. **`getPresets()` / `setPresets(presets)`**: Чтение/запись словаря пресетов в `localStorage` (ключ `timetable_presets`).
61. **`refreshPresetCards()`**: Отображает пресеты как карточки с метаданными (неделя, место, пары, дни, фильтры, количество compare-слотов).
62. **`savePreset()`**: Запрашивает имя и сохраняет `gatherFullState()` в `localStorage`.
63. **`loadPreset(name)`**: Загружает пресет и применяет через `applyFullState()`.
64. **`deletePreset(name)`**: Удаляет пресет после подтверждения.

### Ключевые функции — Шеринг
65. **`encodeResults()`**: Кодирует текущее состояние (prediction results + compare-слоты + настройки) в compact base64url-строку. Формат payload v2: `{v:2, w, cfg, r?, cmp?}`. Поддерживает экспорт даже без prediction (только compare). Результаты `r` содержат агрегированные счётчики (day, slot, totalScore, moreLikely.length и т.д.).
66. **`decodeResults(code)`**: Декодирует base64url-строку обратно в payload-объект.
67. **`exportShareCode()`**: Генерирует код и копирует в буфер обмена.
68. **`importShareCode()`**: Декодирует вставленный код, восстанавливает prediction results (как синтетические `_imported` объекты), compare-слоты (с пересчётом `recomputeCompareResults` если есть группы), и перерисовывает тепловую карту.

### Ключевые функции — URL-хеш
69. **`saveHash()`**: Сериализует полное состояние в `location.hash`: mode, group (schedule), filters, week, pred controls, compare-слоты (`cmp`). Вызывается при каждом изменении состояния (фильтры, неделя, prediction, compare).
70. **`restoreFromHash()`**: Парсит `location.hash`, восстанавливает filters, week, prediction controls, compare-слоты. Если есть `state.pred` — запускает `runPrediction()`. Поддерживает legacy-формат (plain group ID).

### Утилиты
71. **`esc(s)`**: Экранирование HTML-сущностей.
72. **`saveHash()` / `restoreFromHash()`**: Сериализация/десериализация состояния в URL-хэш.
73. **`degreeMatches(filterDeg, groupDeg)`**: Сопоставляет значение фильтра ступени со ступенью группы. Если фильтр = `COMBINED_BAK_SPEC` («Бакалавриат / Специалитет»), соответствует обеим ступеням.

## Алгоритм скоринга (`scoreGroup`)

Подробная документация: [ALGORITHM_DOCS.md](ALGORITHM_DOCS.md), [ALGORITHM_PSEUDOCODE.md](ALGORITHM_PSEUDOCODE.md).

### Константы

```js
W_PROXIMITY  = 3.0    // вес фактора близости
W_LOCATION   = 3.0    // вес фактора корпуса
W_SANDWICH   = 2.0    // вес фактора «бутерброда»
W_DEPARTURE  = 2.0    // вес штрафа за отъезд
BUCKET_THRESHOLD = 4.0 // порог «скорее придёт»
PROXIMITY_DECAY = { 1: 1.0, 2: 0.55, 3: 0.25, 4: 0.1 } // зазор ≥ 5 → 0
LATE_HOUR_FACTOR = { 1–4: 1.0, 5: 0.9, 6: 0.55, 7: 0.25 }
TIREDNESS_TABLE  = { ≤1: 1.0, 2: 0.95, 3: 0.9, 4: 0.85, ≥5: 0.8 }
```

### Этапы (на каждую группу × кандидатный слот)

1. **Конфликт** — обязательное занятие → `have_class`; факультатив → мягкий конфликт (штраф −1.0).
2. **Пустой день** — нет занятий → `less_likely`, score = 0.
3. **Присутствие на кампусе** — множитель `campusFactor`: ≥2 якорных → ×1.0, 1 → ×0.6, только онлайн → ×0.15, только вне → ×0.1.
4. **Близость (P)** — расстояние до ближайшего кампусного занятия (1→1.0, 2→0.55, 3→0.25, 4→0.1, ≥5→0). Модификатор направления: до первого ×0.6, после последнего ×0.8, между ×1.0.
5. **Корпус (L)** — совпадение корпуса с мероприятием (1.0→тот же, 0.75→НК↔ГК, 0.5→кампус, 0.2→онлайн, 0→вне).
6. **Бутерброд (S)** — мероприятие между двумя занятиями: оба якорных → 1.0, якорь+онлайн → 0.5, иначе → 0.
7. **Штраф отъезда (D)** — следующее занятие вне кампуса: через 1 слот → −2.0, через 2 → −1.0.
8. **Усталость (T)** — накопленная нагрузка от предшествующих пар. `effective_load = пары_до − 0.5 × пропуски`. Множитель: ≤1 → ×1.0, 2 → ×0.95, 3 → ×0.9, 4 → ×0.85, ≥5 → ×0.8.
9. **Вечерний фактор** — снижение доступности транспорта вечером. Слоты 1–4 → ×1.0, 5 → ×0.95, 6 → ×0.85, 7 → ×0.7.
10. **Итого**: `raw = 3P + 3L + 2S + D` (clamped ≥0), `score = raw × campusFactor × tirednessFactor × lateHourFactor − softConflictPenalty`.
11. **Бакет**: score ≥ 4.0 → `more_likely`, иначе → `less_likely`.

### Ранжирование слотов

Все кандидатные `(день, слот)` сортируются по:
1. Суммарный балл (убывание).
2. Количество `more_likely` (убывание).
3. Количество `have_class` (возрастание).

Возвращаются 3 лучших.

`lastPredictionResults` автоматически сбрасывается при:

### Элементы управления
Панель имеет эмодзи-заголовок и секционные метки (`.pred-section-label`): 📍 Место, 🕐 Пары, 📅 Дни.

## Управление состоянием (State Management)

### Переменные состояния (глобальные)

```js
let DATA = null;                    // весь nsu_data.json
let groupIndex = [];                // [{id, label, search}, ...]
let currentGroup = null;            // ID текущей группы
let currentMode = 'schedule';       // 'schedule' | 'heatmap'
let heatFilters = [];               // [{faculty, specialty, degree, year, group, label}]

let heatmapInited = false;          // флаг инициализации контролов
let lastPredictionResults = null;   // Array<{day, slot, totalScore, groupDetails, ...}> | null
```

### Правила
- Состояние синхронизируется с URL через `saveHash()`/`restoreFromHash()`.
- `renderHeatmap()` автоматически восстанавливает подсветку и результаты рекомендации из `lastPredictionResults`.
- `lastPredictionResults` сбрасывается при любом изменении фильтров или смене недели.

## Формат хэша URL (URL Hash Format)

Хэш кодируется как URL-encoded JSON:

```
#%7B"mode":"schedule","group":"24812.1"%7D
#%7B"mode":"heatmap","filters":[{"f":"16"}],"week":"odd","pred":{"loc":"1","m":"3","e":"6","days":["1","2","3","4","5"]}%7D
```

### Поля объекта состояния

| Поле | Тип | Описание |
|------|-----|---------|
| `mode` | string | `"schedule"` или `"heatmap"` |
| `group` | string | ID группы (только в mode=schedule) |
| `filters` | array | Массив фильтров `{f, s, d, y, g}` (только в mode=heatmap) |
| `week` | string | `"odd"` или `"even"` (опционально, по умолчанию `"all"`) |
| `pred` | object | Параметры рекомендации (только если активна) |
| `pred.loc` | string | `"1"` (НК/КПА) или `"2"` (ГК/ЛК) |
| `pred.m` | string | Первый допустимый слот (`"1"`–`"7"`) |
| `pred.e` | string | Последний допустимый слот (`"1"`–`"7"`) |
| `pred.days` | string[] | Допустимые дни (`["1","2","3","4","5"]`) |

При восстановлении из хэша с `pred` + `week` автоматически запускается `runPrediction()`.

## Запуск
1. Убедитесь, что [nsu_data.json](nsu_data.json) находится в той же папке.
2. Запустите `python serve.py`.
3. Откройте `http://localhost:8765/timetable.html`.

### Использование рекомендации слотов
1. Перейдите в режим «Загруженность».
2. Добавьте фильтры (факультет, курс и т.д.) или оставьте пустыми для анализа всех групп.
3. Выберите неделю: «Нч» или «Чт» — появится панель «Рекомендация слотов для мероприятия».
4. Укажите место проведения, допустимый диапазон пар и дни.
5. Нажмите «Рассчитать».
6. Результаты отобразятся под тепловой картой, а лучшие слоты подсветятся в таблице.

### Сохранение настроек как пресета
1. Настройте фильтры, неделю и параметры рекомендации.
2. Нажмите «💾 Сохранить», введите имя.
3. Пресет появится в выпадающем списке.
4. Выберите пресет и нажмите «📂 Загрузить» для восстановления.

### Поделиться результатами
1. После расчёта нажмите «📋 Копировать код» — компактная строка скопируется в буфер.
2. Передайте код другому пользователю.
3. Получатель вставляет код в поле и нажимает «📥 Импорт» — результаты отобразятся (карточки + подсветка тепловой карты, но без детализации групп).
